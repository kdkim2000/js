# PRD — 멀티사이트 크롤러 & 학습 플랫폼

## 1. 배경 및 목적

URL을 등록하면 **임의의 문서/튜토리얼 사이트를 크롤링**하여 사이트별로 데이터를 로컬에 저장하고,
통합 학습 플랫폼에서 오프라인으로 탐색할 수 있는 개인용 멀티사이트 학습 도구다.

관리 UI(`/admin`)에서 크롤 대상 등록·실행·상태 모니터링이 가능하고,
크롤된 각 사이트는 `/sites/{siteId}/` 경로에서 독립적으로 탐색할 수 있다.

## 2. 사용자 및 사용 시나리오

- **주 사용자**: 문서/튜토리얼 사이트를 로컬에서 빠르게 읽으려는 개발자 (단일 사용자 로컬 환경)
- **시나리오 1**: 사이트 등록 — 관리 UI에 URL을 입력해 크롤 대상을 추가하고 크롤링을 시작한다.
- **시나리오 2**: 오프라인 읽기 — 크롤된 사이트를 네트워크 없이 로컬에서 읽는다.
- **시나리오 3**: 멀티사이트 검색 — 특정 사이트 내에서 전문 검색으로 개념을 즉시 찾는다.
- **시나리오 4**: 순서 학습 — 파트→챕터→아티클 순서로 이전/다음 탐색, 읽음 진도 관리.
- **시나리오 5**: 코드 실행 — JavaScript 예제 코드를 브라우저에서 직접 실행한다.

## 3. 기능 요구사항

### 3.1 크롤러 (Crawler)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|

| C-04 | SQLite + FTS5로 전문 검색 인덱스를 생성한다 | P0 |
| C-05 | 사이트 전체 목차(TOC)를 파싱해 JSON으로 저장한다 | P0 |
| C-06 | 사이트별로 `data/sites/{siteId}/`에 독립적으로 데이터를 저장한다 | P0 |
| C-07 | `data/registry.json`으로 등록 사이트 목록과 크롤 상태를 관리한다 | P0 |
| C-08 | 이미 크롤링된 아티클은 스킵한다 (증분 크롤링) | P1 |
| C-09 | 요청 간 딜레이 및 지수 백오프 재시도를 적용한다 | P1 |
| C-10 | 아티클에 포함된 이미지를 로컬에 다운로드한다 | P1 |
| C-11 | 실패한 아티클 목록을 `errors.json`으로 기록한다 | P2 |

## 4. 비기능 요구사항

- **성능**: 아티클 페이지 로드 < 200ms (정적 빌드 기준)
- **크롤링 예의**: 요청 간 최소 1,200ms 딜레이, User-Agent 명시
- **오프라인**: 빌드 이후 외부 네트워크 없이 동작
- **확장성**: 단일 사용자 로컬 환경, 서버 배포 불필요

## 5. 범위 외 (Out of Scope)

- 사용자 계정/인증
- 멀티 사용자
- 원본 사이트 콘텐츠 수정
- 자동 재크롤링 스케줄러 (현재 수동/UI 실행)
- 사이트별 완전 자동 HTML 구조 탐지 (셀렉터는 사용자가 직접 지정 가능)

## 7. 배포 전략 (Option A — 완전 정적 내보내기)

### 선택 근거

| 항목 | 이유 |
|------|------|
| 개인 학습 도구 | 서버 운영 비용 불필요 |
| 콘텐츠 변경 빈도 낮음 | 재배포로 충분히 대응 |
| 크롤·관리는 로컬에서만 | Admin UI는 개발 서버(`npm run dev`)에서만 사용 |
| SQLite FTS5 대체 | Pagefind가 빌드된 HTML을 인덱싱 → 클라이언트 검색 |

### 구현 요약

```
[로컬 개발]
  npm run dev        → SQLite + API Routes 포함 (full feature)
  /admin 크롤 관리   → 개발 서버에서만 동작

[정적 빌드]
  npm run build      → next build (output: 'export') + npx pagefind → out/ 생성

[배포]
  vercel --prod      → out/ 을 Vercel에 정적 배포
  또는 GitHub Actions → 자동 빌드 + 배포
```

### 검색 전략

| 환경 | 검색 방식 |
|------|---------|
| 로컬 개발 | `POST /api/search` → SQLite FTS5 (better-sqlite3) |
| 정적 빌드/배포 | `import('/pagefind/pagefind.js')` → Pagefind 클라이언트 검색 |

### 제약 사항

- API Routes (`/api/**`)는 개발 서버 전용; 정적 배포에서는 Pagefind로 대체
- Admin UI (`/admin`)는 정적 배포에서 "로컬 전용" 안내 표시
- `better-sqlite3`는 빌드 시 SSG에만 사용; 런타임 서버 불필요
- 이미지는 `unoptimized: true` (Next.js Image Optimization API 불필요)
